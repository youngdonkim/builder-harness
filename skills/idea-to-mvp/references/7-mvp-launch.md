이 단계는 **말이 아니라 행동으로 가설을 판정하는** 이 스킬의 최종 통과 기준이다. 인터뷰(통과 기준③)는 면전 호의에 물들지만, 광고 클릭→가입→사용→결제는 거짓말을 못 한다.

**이 단계의 생명은 순서다** — 기준을 광고 집행 *전에* 동결한다. 숫자를 보고 나서 기준을 정하면 창업자 뇌는 무조건 자기한테 유리하게 해석한다(사후 자기합리화). 합격선은 시험 전에 그어놔야 의미가 있다.

**소액 광고의 한계를 알고 쓴다** — 소액 예산의 퍼널 숫자는 광고 소재·타겟팅 실력에 흔들린다 ("광고가 구려서 죽었는지 제품이 필요 없어서 죽었는지" 구분 어려움). 그래서 채널은 반드시 **이중**: 발품(정성 — *왜* 쓰는지/안 쓰는지 들림) + 광고(정량 — 퍼널 숫자). 하나로만 판정하지 않는다.

## 1. 단계 목표

- **`launch-plan.md`** — 광고 집행 *전* 동결: 퍼널 go 선 · 예산 · 측정 기간 · 채널 계획.
- **이중 채널 집행** — 발품(커뮤니티·직접 아웃리치) + 소액 광고. 측정 기간 동안 퍼널 데이터 + 정성 voice 수집.
- **`launch-retro.md`** — 실측 vs 기준 + 정성 종합 → **가설 검증 / 피봇 / 폐기** 최종 판정 (통과 기준④).

**SoT 책임** (SKILL.md §2.1): 런치 기준 = `launch-plan.md` (동결 후 불변), 실측·판정 = `launch-retro.md`. 계측 이벤트 *정의*는 mvp-build.md SoT — 여기서 재정의 금지, 참조만.

## 2. 진행 절차 — 3막 (순서 강제)

### 2.1 막1 — 기준 박기 (광고 집행 전, 강제 잠금)

**launch-plan.md를 쓰고 사용자가 확정하기 전에는 광고 집행 금지.** SKILL.md §4의 건너뛰기 옵션도 이 잠금엔 적용 안 됨.

AI가 초안 모드로 launch-plan 초안을 완주해서 제시 — 단 **go 선 수치는 인터뷰 모드** (사용자 결정 영역, SKILL.md §1.5). AI는 market-research 검증 가설 + mvp-build 이벤트 정의를 base로 *후보 수치 + 업계 통상 범위 + 이유*를 제시하고, 사용자가 정한다.

**박을 4개**:

1. **퍼널 go 선** — mvp-build.md 이벤트 정의 그대로 사용 (재정의 금지):

   | 퍼널 | 측정 | go 선 | 이유 |
   |---|---|---|---|
   | 유입→가입 | 랜딩 전환율 | __% | (후보 제시 시 통상 범위 병기) |
   | 가입→핵심행동 | 활성화율 | __% | |
   | 재방문 | 7일 내 재방문율 | __% | |
   | 지불 | 결제 전환율 (또는 프록시) | __% | |

   모든 선이 넘어야 go는 아님 — **어떤 조합이면 go / 어떤 조합이면 피봇 검토**인지 판정 룰도 함께 박는다 (예: "가입·활성화 통과 + 지불 미달 = 가격 피봇 검토"). 표본이 작으므로 판정 룰은 단순하게.

2. **예산·기간** — 광고비 상한 (예: 30만원) + 측정 기간 (예: 2주). 기간 전 조기 종료 조건(예산 소진·명백한 무반응)도 명시.

3. **채널 계획** —
   - **발품**: 어디 커뮤니티·어떤 방식(글·DM)·목표 접촉 수. 정성 voice 수집 방법 (댓글·DM 대화 기록).
   - **광고**: 매체(예: Meta·네이버)·타겟·소재 방향. 채널 구분 파라미터(mvp-build §2.4) 확인.

4. **동결 선언** — "이 기준은 집행 후 변경하지 않는다. 변경이 불가피하면 *변경 전에* 사유를 launch-plan.md 하단에 기록한다." frontmatter `status: frozen` + 동결 날짜.

### 2.2 막2 — 이중 채널 집행 + 수집

**광고 소재·랜딩 카피는 초안 모드로 생산** — AI가 소재 후보 여러 벌 완주 (카피는 `ux-writing-reviewer` 에이전트 통과). 집행(광고 계정 세팅·게시)은 사용자가.

**발품도 계획대로** — 커뮤니티 글·DM은 AI가 초안, 게시·대화는 사용자. 대화에서 나온 정성 voice는 그때그때 기록 (launch-retro 입력).

**측정 기간 중 룰**:

- **퍼널에 영향 주는 변경 금지** — 랜딩 카피 수정·가격 변경·기능 추가는 측정을 오염시킨다. 발견한 개선 아이디어는 `추후 결정:`으로 기록만.
- **명백한 버그 fix는 OK** — 퍼널이 *깨진* 것(결제 에러·가입 불능)은 즉시 수정하고 수정 시점을 기록 (판정 시 참작).
- AI는 주기적으로 퍼널 중간 숫자를 정리해 보여줄 수 있으나, **중간 숫자로 기준을 흔들지 않는다** — go 선 재론은 금지.

### 2.3 막3 — launch-retro (통과 기준④, 최종 판정)

측정 기간 종료 후:

1. **AI가 판정 자료를 초안 모드로 완주** — 퍼널 실측 vs go 선 표 (채널 구분 포함) + 발품 정성 voice 종합 + 판정 룰 적용 결과 + 버그 fix·특이사항 타임라인.
2. **판정은 사람** (SKILL.md §1.5 사람 몫 불변). 3분기:

   - **가설 검증 (go)** → **production 졸업** — 이 하네스의 범위 밖 (매출 스케일업·운영 자동화·다채널 확장은 분기별 KPI·운영 도구 영역). 7단계 전부 완주. launch-retro에 production 입력(검증된 채널·남긴 `추후 결정` 목록) 정리.
   - **피봇** → 무엇이 죽고 무엇이 살았는지로 되돌아갈 단계 결정 (예: 지불만 미달 = 가격·과금 모델 손질 후 재런치 / 활성화 미달 = 핵심행동 UX 재설계 → 3~4단계(ScreenDesign·Prototype) / 문제 자체가 흔들림 = 사실상 폐기에 가까움). 영향 범위 처리 원칙은 SKILL.md §1.4 "이전 결정 변경 처리".
   - **폐기 (no-go)** → 이 아이디어는 닫음. 학습을 launch-retro에 보존. 새 아이디어는 새 프로젝트에서 처음부터.

**Use·Love·Pay 매핑** (판정 자료에 이 3축으로도 요약):

| 축 | 측정 |
|---|---|
| Use (쓰는가) | 활성화율 + 재방문율 |
| Love (좋아하는가) | 발품 정성 voice (자발 추천·재문의) — 표본이 충분히 쌓이면 Sean Ellis 설문("없어지면 매우 실망" 40%)을 *보조*로 |
| Pay (지불하는가) | 실결제 전환 (또는 지불 프록시) |

Sean Ellis 40%는 소액 광고 표본으론 통계가 안 서는 경우가 많다 — **주 판정은 launch-plan의 퍼널 go 선**, Sean Ellis는 표본 될 때만 보조.

## 5. 좋은 예 vs 나쁜 예

- **기준 박기** — 좋은: "가입 전환 5%·활성화 40%·재방문 20%, 판정 룰까지 집행 3일 전 동결". / 나쁜: "일단 광고 돌려보고 숫자 보고 판단하자" (사후 기준 = 자기합리화 문).
- **채널** — 좋은: "광고 가입 31명(전환 4.2%) + 발품 대화 14건에서 '중개사 말을 못 믿겠다' 7회 반복". / 나쁜: 광고 숫자만 놓고 "전환 낮음 = 폐기" (소재 문제인지 제품 문제인지 미구분).
- **측정 기간 대응** — 좋은: "결제 에러 발견 즉시 fix + 타임라인 기록, 랜딩 카피 개선안은 추후 결정으로 보류". / 나쁜: 중간 숫자 보고 랜딩 카피 교체 (측정 오염).
- **판정** — 좋은: "가입·활성화 통과, 지불 미달 → 판정 룰대로 가격 피봇 검토, 근거 표 첨부". / 나쁜: "느낌상 반응 나쁘지 않으니 go".

## 6. 사용자 응대 톤 + 코칭

- **톤**: SKILL.md §1.3대로 반말·친근·짧게.
- **진입 안내**: "이제 진짜 시장에 내보내서 검증할 차례야. 순서가 중요해 — 광고 켜기 *전에* '얼마면 합격'인지 선부터 긋자. 내가 퍼널 기준 초안(후보 수치+통상 범위+이유)을 만들어올게, 수치는 네가 정해."
- **go 선 코칭**: 사용자가 수치를 못 정하면 후보 2~3개 + 통상 범위 제시 (§1.4). "모르겠으면 보수적인 쪽으로 — 낮은 선으로 통과하는 것보다 높은 선에 못 미치는 걸 아는 게 싸."
- **동결 후 흔들림 대응**: 측정 중 "기준 좀 낮출까?" 신호 — "그게 이 단계가 막으려는 바로 그거야. 낮출 이유가 진짜면 *변경 이력에 사유 먼저* 쓰고 바꾸자 — 근데 웬만하면 기간 끝까지 보고 판정에서 얘기하자."
- **판정 안내**: 자료를 표로 보여주고 — "판정은 네 몫이야. 자료 기준으로는 [요약]. 어느 쪽?"
