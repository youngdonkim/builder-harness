# 마크다운 목차(TOC) 작성 규칙

여러 하위 섹션을 `·`(가운뎃점) 같은 구분자로 한 줄에 나열하지 않는다 — 줄바꿈이 없어 구조가 한눈에 안 들어온다. 대신 **들여쓰기된 중첩 리스트**로 펼친다.

**나쁜 예**:
```
- §2 진행 절차 — §2.2 플로우 골격 · §2.3 화면 명세(7요소) · §2.4 시나리오 경로
```

**좋은 예**:
```
- §2 진행 절차
  - §2.2 플로우 골격
  - §2.3 화면 명세(7요소)
  - §2.4 시나리오 경로
```

하위 절이 한 단계 더 깊어지면(예: §2.3.1) 들여쓰기도 그만큼 더 들어간다.

**`§1`·`§2.3` 같은 절 번호는 마크다운 문법이 아니다** — 문서 본문에서 다른 절을 텍스트로 교차 참조할 때 쓰는 이 저장소만의 관례다. 실제 헤더는 `#`·`##`·`###`로 따로 존재하므로, 목차 항목을 실제 헤더 문법으로 쓰지 않는다 — 그러면 같은 이름의 헤더가 중복돼 문서 아웃라인이 꼬인다.
